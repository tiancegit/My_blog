一道关于字符串的面试题

作者：xlzd | 时间：2016-03-30 21:16:00 | 分类：技术 | 评论：5 评论

今天的题目是：编写一个算法，实现基本的字符串“压缩”算法，比如对于字符串abbbbbccccdddcccccc，经过算法处理之后得到的输出为a1b5c4d3c6，如果处理后的字符串长度不小于原串长度，则返回原串。

这个问题乍一看非常容易，然后可以分分钟撸出代码来了：

def compress_str(string):
    result = ''
    count = 1
    current = string[0]

    for ch in string[1:]:
        if ch == current:
            count += 1
        else:
            result += current + str(count)
            count = 1
            current = ch

    result += current + str(count)

    if len(result) >= len(string):
        return string
    return result

这段代码确实可以解决上面的问题，不过效率却非常低：频繁的字符串拼接是一个非常费时耗内存的操作，而上面的代码中，会经过n次迭代(n为字符串长度)和k次字符串的拼接(k为string中字符的数量)。在代码编写过程中，我们应该减少字符串拼接的操作(我认识一个人，通过拼接字符串来凑一个一百多个属性的json字符串，让我看到他的代码的时候，我想杀了他)。

所以，我们可以对上面的代码做一些优化，在Python中，我们可以将中间结果保存到一个list中，在Java中，则可以更加方便的使用StringBuilder/StringBuffer。优化后的代码如下：

def compress_str(string):
    result = []
    count = 1
    current = string[0]

    for ch in string[1:]:
        if ch == current:
            count += 1
        else:
            result.append(current)
            result.append(str(count))
            count = 1
            current = ch

    result.append(current)
    result.append(str(count))

    if len(result) >= len(string):
        return string
    return ''.join(result)

经过上面的修改之后，我们只需要一次字符串拼接操作就可以得到结果了，虽然中间使用了一个list，但是这是值得的，要知道，字符串使用起来确实很方便，不过带来的性能损耗却非常严重。代码到了这一步，还可以再进一步优化吗？我们注意题目中有这样一句话：

如果处理后的字符串长度不小于原串长度，则返回原串。

由于对一个序列的迭代操作比对list的多次append操作代价要小得多，所以，我们可以在往list塞中间数据之前，先判断一下有没有必要(处理后的结果是否比原串短)做这些操作。

def check(string):
    if not string:
        return 0

    size = 2
    current = string[0]

    for ch in string[1:]:
        if ch != current:
            size += 2
            current = ch
    return size

def compress_str(string):
    if check(string) >= len(string):
        return string
    result = []
    count = 1
    current = string[0]

    for ch in string[1:]:
        if ch == current:
            count += 1
        else:
            result.append(current)
            result.append(str(count))
            count = 1
            current = ch
    result.append(current)
    result.append(str(count))
    return ''.join(result)

经过上面的修改，现在，我们在分解字符串之前，会先判断这个操作是否值得，如果这不是必须的，则会直接返回原始字符串。总之，虽然字符串很方便使用，但是其修改的代价却有点大， 如果可能，我们还是尽可能减少对字符串的修改操作吧，尤其是当需要对一个大字符串做操作的时候。

好了，今天就到这里吧~~~
